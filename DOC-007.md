# 6. IMAGE STORAGE, PROCESSING, & OPTIMIZATION PIPELINE

## 6.1 Direct S3 Pre-Signed Upload Flow
To optimize server performance, the platform routes file uploads directly to AWS S3 using secure pre-signed URLs, bypassing the core application server.

```
[ Client ] --( requests pre-signed URL )-----> [ Application API ]
                                                      |
                                                      v
[ Client ] <--( returns pre-signed S3 URL )-- [ Validate session ]
    |
    v
[ Uploads file binary directly to S3 ]
    |
    v
[ S3 triggers Lambda to optimize image and save webp format ]
```

### Upload Execution Steps:
1. **Pre-signed URL Request:** The client sends an authorization request to the core API (`POST /api/v1/assets/presigned-url`) specifying the file's target content type, file size, and destination directory.
2. **Access Control Check:** The API validates that the user is authorized to upload assets. It checks that the file size is within limits (e.g., max 5MB for product photos) and matches allowed MIME types (`image/jpeg`, `image/png`, `image/webp`).
3. **URL Generation:** The API requests a secure pre-signed URL from AWS S3 using restricted IAM credentials. The URL is configured with a short expiration TTL (exactly 5 minutes) and is returned to the client.
4. **Direct Binary Upload:** The client uploads the image binary directly to S3 using the pre-signed URL via an HTTP PUT request.

## 6.2 Storage Topology & Folder Hierarchy
The AWS S3 asset bucket structure is organized into separate directories to support multi-tenant isolation:

```
s3://zayjar-assets-production/
  ├── global/
  │    └── placeholders/
  └── tenants/
       ├── tenant_uuid_1/
       │    ├── branding/
       │    │    ├── logo.webp
       │    │    └── banner.webp
       │    └── branches/
       │         └── branch_uuid_1/
       │              ├── categories/
       │              └── products/
       │                   ├── product_uuid_a.webp
       │                   └── product_uuid_b.webp
       └── tenant_uuid_2/
```

- **Branding Directory:** Stores brand-specific assets (such as logos and background banners) linked directly to the tenant's profile.
- **Product Directory:** Stores product photos organized by branch and product ID. This separation ensures assets are logically segmented and easy to clean up when a product or branch is deleted.

## 6.3 Real-Time Serverless Image Optimization Engine
When an image is uploaded to S3, a serverless lambda worker processes and optimizes the file automatically:

- **S3 Upload Trigger:** S3 triggers an AWS Lambda worker when a new file is uploaded to the raw staging directory.
- **Image Processing Optimization:** The Lambda worker downloads the raw image and processes it using the high-performance **Sharp** image processing library:
  - Converts the image format to **WebP** to maximize compression.
  - Resizes the image to standardized dimensions depending on the image type (e.g., logo resized to 512x512, product photos resized to 1024x768).
  - Optimizes compression settings, targeting a quality level of 80% to balance file size and visual clarity.
- **Optimized Storage:** The Lambda worker writes the optimized WebP image back to the production S3 directory and deletes the raw staging image to clean up storage.

## 6.4 CloudFront CDN Edge Caching & Cache Invalidation Strategy
To speed up asset delivery worldwide, the platform caches optimized images on edge servers using AWS CloudFront:

- **Edge Caching Policy:** CloudFront caches images at edge locations globally. Assets are configured with an aggressive caching policy using a long expiration TTL (exactly 1 year): `Cache-Control: public, max-age=31536000, immutable`.
- **Query Parameter Cache Invalidation:** To update cached images without manual invalidations, the platform uses a query parameter cache-busting strategy. When an image is updated, the database updates the image's timestamp, and the client loads the image using a cache-busting URL parameter (e.g., `/product_uuid.webp?v=1787161920`), forcing CloudFront to fetch and cache the updated asset.

## 6.5 Asset Validation & Upload Restrictions
To protect against malicious files, all uploads must pass validation checks:
- **Maximum File Sizes:**
  - Tenant Logos: 2 MB maximum.
  - Tenant Banners: 4 MB maximum.
  - Product Photos: 5 MB maximum.
- **Allowed Formats:** The S3 bucket policy blocks uploads that do not match allowed file formats (`image/jpeg`, `image/png`, `image/webp`).
- **EXIF Metadata Stripping:** The image optimization pipeline strips metadata and EXIF data (such as GPS coordinates and camera models) from uploaded photos to protect user privacy.
