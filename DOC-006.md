# 5. SECURITY IMPLEMENTATION SPECIFICATION

## 5.1 JWT Structure & Cryptographic Signing
Authentication is stateless and uses high-security JSON Web Tokens signed with an asymmetric **RS256** algorithm.
- **Asymmetric Encryption Key Management:** The authentication service generates a 2048-bit RSA key pair. The private key is stored securely in AWS Secrets Manager and used exclusively for signing tokens. The public key is distributed across microservices and gatekeepers, enabling them to verify tokens locally without hitting the central authentication database.
- **Access Token Payload Fields:**
  ```json
  {
    "sub": "u410c-9a1b-42b8-bf83-097a18fcd34c",
    "tenantId": "7a18f-39b0-4050-bf83-097a18fcd34b",
    "branchId": "b020c-9a1b-42b8-bf83-097a18fcd342",
    "email": "owner@restaurantbrand.com",
    "roles": ["RESTAURANT_OWNER"],
    "permissions": ["menu:create", "menu:update", "orders:read"],
    "iat": 1787161920,
    "exp": 1787162820,
    "iss": "https://auth.zayjar.com"
  }
  ```

## 5.2 Session Management & Sliding Window Revocation
The platform uses a hybrid session management approach to support fast stateless token validation while allowing for immediate session revocation when needed:
- **Stateless Token Verification:** API endpoints verify the asymmetric signature of incoming access tokens locally, avoiding database roundtrips.
- **Blacklist Revocation Engine:** Active revocations (such as user logouts, password resets, or security events) are tracked in a high-speed Redis cluster. When an access token is revoked, its unique ID (`jti`) is written to the Redis blacklist with a TTL matching the token's remaining lifespan. The API Gateway queries the Redis blacklist on every incoming request to reject blacklisted tokens.
- **Sliding Window Refresh Tokens:** Refresh tokens are tracked in both PostgreSQL and Redis. Every time a refresh token is used to renew an access token, the system rotates the refresh token: it invalidates the used token, generates a new one with a sliding-window expiration, and returns it to the client in a secure, HttpOnly cookie.

## 5.3 Cross-Site Request Forgery (CSRF) Mitigation
The API protects against Cross-Site Request Forgery (CSRF) attacks with robust security policies:
- **HttpOnly Cookie Attributes:** Refresh tokens are stored in cookies configured with strict security flags: `HostOnly; Secure; SameSite=Strict; Path=/; Domain=.zayjar.com`.
- **Double-Submit Cookie Patterns:** For administrative and user actions, the API Gateway requires the client to include a custom header `X-CSRF-Token` on all mutating requests (`POST`, `PUT`, `DELETE`). The gateway validates that the header value matches the CSRF token stored in the user's active session, rejecting requests with mismatched or missing tokens.

## 5.4 Cross-Site Scripting (XSS) Sanitization Pipelines
The platform applies comprehensive sanitization measures to prevent Cross-Site Scripting (XSS) vulnerabilities:
- **Strict Content Security Policy (CSP):** The web portals enforce strict Content Security Policy headers, restricting resource loading to trusted domains and blocking inline scripts, style injections, and unauthorized object executions:
  ```http
  Content-Security-Policy: default-src 'self'; script-src 'self' https://js.stripe.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; img-src 'self' data: https://cdn.zayjar.com; frame-src 'self' https://js.stripe.com;
  ```
- **Input Sanitization Filters:** Global NestJS middleware processes all incoming JSON payloads. It uses the `dompurify` and `class-sanitizer` libraries to sanitize HTML characters and strip malicious scripts before storing data in the database.

## 5.5 SQL Injection Defenses & ORM Boundaries
The system prevents SQL Injection attacks by restricting database access and routing all queries through an Object-Relational Mapping (ORM) boundary:
- **Parameterized Queries:** Relational operations are executed using **Prisma ORM**. Prisma handles database parameterization automatically, ensuring user inputs are treated as literal values rather than executable code.
- **Raw SQL Queries Restrictions:** The use of raw SQL queries is restricted. In rare cases where raw SQL is required for advanced reporting, developers must use Prisma's safe parameterized query methods (such as `prisma.$queryRaw`) to protect against injection vulnerabilities.

## 5.6 Distributed Rate Limiting via Redis Token Bucket
To protect against brute-force attacks and resource exhaustion, the API Gateway enforces rate limiting using a Redis-backed token bucket algorithm:
- **Rate Limit Tiers:**
  - Standard Public Endpoints (e.g., browsing menus): 120 requests per minute per IP.
  - Checkout & Order Submissions: 30 requests per minute per IP.
  - Authentication and Login Endpoints: 10 requests per minute per IP.
- **Redis Implementation:** The gateway tracks request limits in a Redis cache cluster. If a client exceeds their rate limit, the gateway blocks the request and returns an HTTP 429 Too Many Requests response with a standard retry-after header.

## 5.7 Immutable Structured Audit Logs
To support security compliance, tracking, and auditing, critical system operations write immutable records to the `audit_logs` table:
- **Log Archival Pipeline:** Every administrative mutation (including product updates, order cancellations, and configuration changes) triggers a background worker that writes detailed change records to the `audit_logs` table.
- **Immutable Storage:** The `audit_logs` table is configured with write-only permissions: records can be inserted, but `UPDATE` and `DELETE` operations are blocked at the database level. For long-term security, audit logs are periodically streamed to an external write-once-read-many (WORM) storage system.

## 5.8 Encryption standards (Transit, Rest, Application Level)
The platform applies industry-standard encryption standards to secure data across its lifecycle:
- **Data in Transit:** All network traffic is encrypted using TLS 1.3. Unencrypted HTTP requests are automatically redirected to HTTPS at the load balancer.
- **Data at Rest:** Persistent storage is encrypted at rest. AWS RDS database volumes, Redis caches, and S3 asset buckets use AES-256 encryption managed by AWS Key Management Service (KMS).
- **Application-Level Hashing:** User passwords are encrypted using the **Argon2id** hashing algorithm, protecting credentials against offline brute-force attacks.

## 5.9 Enterprise Secrets Management
To maintain system security, sensitive variables and secrets are managed through a centralized secret store:
- **Centralized Secrets Store:** Secrets (including API keys, private signing keys, database passwords, and provider credentials) are stored securely in **AWS Secrets Manager**.
- **Dynamic Injections:** Application containers fetch and inject required credentials dynamically into memory at startup, ensuring sensitive credentials are never stored or hardcoded in source repositories.

## B.1 Multi-Tenant Context Middleware

The following middleware extracts the tenant ID from the subdomain or custom domain on every incoming request, validates it against a Redis cache, and sets it in a thread-local async execution context.

```typescript
import { Injectable, NestMiddleware, HttpStatus, HttpException } from '@nestjs/common';
import { Request, Response, NextFunction } from 'express';
import { createClient } from 'redis';
import { AsyncLocalStorage } from 'async_local_storage';
import { PrismaService } from '../prisma/prisma.service';

// Instantiating thread-safe context storage
export const tenantContextStorage = new AsyncLocalStorage<{ tenantId: string }>();

@Injectable()
export class TenantContextMiddleware implements NestMiddleware {
  private redisClient;

  constructor(private readonly prisma: PrismaService) {
    // Initializing high-speed caching redis connection
    this.redisClient = createClient({ url: process.env.REDIS_URL });
    this.redisClient.connect().catch(console.error);
  }

  async use(req: Request, res: Response, next: NextFunction) {
    const host = req.headers.host || '';
    const correlationId = req.headers['x-correlation-id'] || `req-${Math.random().toString(36).substr(2, 9)}`;
    req.headers['x-correlation-id'] = correlationId;

    let tenantId: string | null = null;

    try {
      // 1. Check for manual override header (authorized developer scoping)
      const headerTenantId = req.headers['x-tenant-id'];
      if (headerTenantId && typeof headerTenantId === 'string') {
        tenantId = headerTenantId;
      } else {
        // 2. Extract subdomain or custom domain mapping
        const parsedHost = host.split(':')[0]; // Strip port if present
        
        if (parsedHost.endsWith('.zayjar.com')) {
          const subdomain = parsedHost.replace('.zayjar.com', '');
          // Resolve subdomain from cache first
          tenantId = await this.redisClient.get(`subdomain:${subdomain}`);
          
          if (!tenantId) {
            // Cache miss: query PostgreSQL database
            const tenant = await this.prisma.tenant.findUnique({
              where: { subdomain, deletedAt: null },
              select: { id: true }
            });
            if (tenant) {
              tenantId = tenant.id;
              // Write back to cache with 1-hour expiration
              await this.redisClient.set(`subdomain:${subdomain}`, tenantId, { EX: 3600 });
            }
          }
        } else {
          // Resolve verified custom domain mapping
          tenantId = await this.redisClient.get(`domain:${parsedHost}`);
          
          if (!tenantId) {
            const tenant = await this.prisma.tenant.findUnique({
              where: { customDomain: parsedHost, deletedAt: null },
              select: { id: true }
            });
            if (tenant) {
              tenantId = tenant.id;
              await this.redisClient.set(`domain:${parsedHost}`, tenantId, { EX: 3600 });
            }
          }
        }
      }

      if (!tenantId) {
        throw new HttpException(
          {
            statusCode: HttpStatus.NOT_FOUND,
            timestamp: new Date().toISOString(),
            path: req.originalUrl,
            error: 'Tenant Not Found',
            message: 'Unable to resolve the active tenant domain context.',
            correlationId
          },
          HttpStatus.NOT_FOUND
        );
      }

      // 3. Inject resolved tenant context into thread local storage
      tenantContextStorage.run({ tenantId }, () => {
        // Bind dynamic properties to the request context
        req['tenantId'] = tenantId;
        next();
      });

    } catch (error) {
      if (error instanceof HttpException) {
        return res.status(error.getStatus()).json(error.getResponse());
      }
      return res.status(HttpStatus.INTERNAL_SERVER_ERROR).json({
        statusCode: HttpStatus.INTERNAL_SERVER_ERROR,
        timestamp: new Date().toISOString(),
        path: req.originalUrl,
        error: 'Internal Server Error',
        message: 'A fatal exception occurred during tenant domain scoping.',
        correlationId
      });
    }
  }
}
```

## B.2 Role-Based Access Control (RBAC) & Tenant Gating Guard

The following guard enforces role assignments and resource permissions, ensuring that users can only operate on data scoping to their authorized tenant workspace.

```typescript
import { Injectable, CanActivate, ExecutionContext, HttpStatus, HttpException } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import * as jwt from 'jsonwebtoken';

@Injectable()
export class RbacPermissionGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest();
    const correlationId = request.headers['x-correlation-id'];

    // Retrieve handler-level metadata permissions tags
    const requiredPermission = this.reflector.get<string>('permission', context.getHandler());
    if (!requiredPermission) {
      return true; // Unprotected route
    }

    const authHeader = request.headers.authorization;
    if (!authHeader || !authHeader.startsWith('Bearer ')) {
      throw new HttpException(
        {
          statusCode: HttpStatus.UNAUTHORIZED,
          timestamp: new Date().toISOString(),
          path: request.originalUrl,
          error: 'Unauthorized',
          message: 'Access token was not provided or has expired.',
          correlationId
        },
        HttpStatus.UNAUTHORIZED
      );
    }

    const token = authHeader.split(' ')[1];
    
    try {
      // Validate signature against the RSA Public Key injected at runtime
      const publicKey = process.env.JWT_PUBLIC_KEY;
      const decodedPayload = jwt.verify(token, publicKey, { algorithms: ['RS256'] }) as any;
      
      request.user = decodedPayload;

      // Ensure user tenant context matches active domain tenant context
      if (decodedPayload.tenantId && decodedPayload.tenantId !== request.tenantId) {
        throw new HttpException(
          {
            statusCode: HttpStatus.FORBIDDEN,
            timestamp: new Date().toISOString(),
            path: request.originalUrl,
            error: 'Forbidden',
            message: 'Cross-tenant access attempt was detected and blocked.',
            correlationId
          },
          HttpStatus.FORBIDDEN
        );
      }

      // Check if user holds the platform owner administrator master key
      if (decodedPayload.roles.includes('PLATFORM_OWNER')) {
        return true;
      }

      // Evaluate permission string matching
      const hasPermission = decodedPayload.permissions.includes(requiredPermission);
      if (!hasPermission) {
        throw new HttpException(
          {
            statusCode: HttpStatus.FORBIDDEN,
            timestamp: new Date().toISOString(),
            path: request.originalUrl,
            error: 'Forbidden',
            message: `Your active credentials lack the required permission: [${requiredPermission}].`,
            correlationId
          },
          HttpStatus.FORBIDDEN
        );
      }

      return true;

    } catch (error) {
      if (error instanceof HttpException) {
        throw error;
      }
      throw new HttpException(
        {
          statusCode: HttpStatus.UNAUTHORIZED,
          timestamp: new Date().toISOString(),
          path: request.originalUrl,
          error: 'Unauthorized',
          message: 'The provided access token was invalid or has been blacklisted.',
          correlationId
        },
        HttpStatus.UNAUTHORIZED
      );
    }
  }
}
```

## B.3 Transactional Order Placement & Pricing Engine Service

The following NestJS service orchestrates order placement in a single database transaction, applying price inheritance rules, calculating taxes dynamically, updating table statuses, and broadcasting updates to kitchen displays.

```typescript
import { Injectable, HttpStatus, HttpException } from '@nestjs/common';
import { PrismaService } from '../prisma/prisma.service';
import { KdsGateway } from '../kds/kds.gateway';
import { CreateOrderDto } from './dto/create-order.dto';

@Injectable()
export class OrderService {
  constructor(
    private readonly prisma: PrismaService,
    private readonly kdsGateway: KdsGateway
  ) {}

  async checkout(tenantId: string, dto: CreateOrderDto): Promise<any> {
    const correlationId = `ord-tx-${Math.random().toString(36).substr(2, 9)}`;

    // Execute the complete order assembly inside a managed database transaction
    return this.prisma.$transaction(async (tx) => {
      
      // 1. Fetch branch configuration settings
      const branch = await tx.branch.findUnique({
        where: { id: dto.branchId, tenantId, deletedAt: null },
        include: { restaurant: true }
      });
      if (!branch) {
        throw new HttpException('Target branch was not found under this tenant profile.', HttpStatus.NOT_FOUND);
      }

      // 2. Validate table occupancy rules for DINE_IN checkouts
      if (dto.type === 'DINE_IN' && dto.tableId) {
        const table = await tx.table.findUnique({
          where: { id: dto.tableId, branchId: dto.branchId, deletedAt: null }
        });
        if (!table) {
          throw new HttpException('Target dining table does not exist in this branch.', HttpStatus.NOT_FOUND);
        }
        if (table.status === 'OCCUPIED') {
          throw new HttpException('The selected table is occupied.', HttpStatus.CONFLICT);
        }
        // Lock table occupancy state
        await tx.table.update({
          where: { id: dto.tableId },
          data: { status: 'OCCUPIED' }
        });
      }

      let orderSubtotal = 0;
      const orderItemsToCreate = [];

      // 3. Process shopping cart items sequentially
      for (const cartItem of dto.items) {
        const product = await tx.product.findUnique({
          where: { id: cartItem.productId, tenantId, deletedAt: null }
        });
        if (!product || !product.isAvailable) {
          throw new HttpException(`Product with ID [${cartItem.productId}] is currently unavailable.`, HttpStatus.UNPROCESSABLE_ENTITY);
        }

        let itemUnitPrice = Number(product.basePrice);

        // A. Evaluate sizing adjustments
        if (cartItem.sizeId) {
          const size = await tx.productSize.findUnique({
            where: { id: cartItem.sizeId, productId: product.id }
          });
          if (!size) {
            throw new HttpException(`Size modifier [${cartItem.sizeId}] is invalid for this product.`, HttpStatus.UNPROCESSABLE_ENTITY);
          }
          itemUnitPrice += Number(size.priceAdjustment);
        }

        // B. Evaluate variant price overrides
        if (cartItem.variantId) {
          const variant = await tx.productVariant.findUnique({
            where: { id: cartItem.variantId, productId: product.id }
          });
          if (!variant) {
            throw new HttpException(`Variant [${cartItem.variantId}] is invalid for this product.`, HttpStatus.UNPROCESSABLE_ENTITY);
          }
          if (variant.stockQuantity < cartItem.quantity) {
            throw new HttpException(`Insufficient inventory for variant: ${variant.name}.`, HttpStatus.CONFLICT);
          }
          itemUnitPrice = Number(variant.price); // Direct override

          // Decrement variant stock inventory
          await tx.productVariant.update({
            where: { id: cartItem.variantId },
            data: { stockQuantity: { decrement: cartItem.quantity } }
          });
        }

        let itemAddonsTotal = 0;
        const addonsToCreate = [];

        // C. Evaluate addon modifications
        if (cartItem.addons && cartItem.addons.length > 0) {
          for (const addonSelection of cartItem.addons) {
            const addonItem = await tx.addonItem.findUnique({
              where: { id: addonSelection.addonItemId, isAvailable: true }
            });
            if (!addonItem) {
              throw new HttpException(`Selected addon customization is unavailable.`, HttpStatus.UNPROCESSABLE_ENTITY);
            }
            itemAddonsTotal += Number(addonItem.price);
            addonsToCreate.push({
              tenantId,
              addonItemId: addonItem.id,
              price: addonItem.price
            });
          }
        }

        const calculatedLinePrice = (itemUnitPrice + itemAddonsTotal) * cartItem.quantity;
        orderSubtotal += calculatedLinePrice;

        orderItemsToCreate.push({
          tenantId,
          productId: product.id,
          sizeId: cartItem.sizeId || null,
          variantId: cartItem.variantId || null,
          quantity: cartItem.quantity,
          unitPrice: itemUnitPrice,
          totalPrice: calculatedLinePrice,
          cookingStatus: 'PENDING',
          orderItemAddons: {
            create: addonsToCreate
          }
        });
      }

      // 4. Run tax and financial ledger calculations
      const taxRate = Number(branch.restaurant.taxPercentage) / 100;
      const taxAmount = Number((orderSubtotal * taxRate).toFixed(2));
      const orderTotal = Number((orderSubtotal + taxAmount).toFixed(2));

      // 5. Generate human-friendly order reference number
      const sequenceValue = await tx.$queryRawUnsafe<any>(`SELECT nextval('order_seq_${branch.id}') as val`);
      const orderNumber = `ORD-${new Date().getFullYear()}-${sequenceValue[0].val.toString().padStart(5, '0')}`;

      // 6. Write final Order to database
      const order = await tx.order.create({
        data: {
          tenantId,
          branchId: dto.branchId,
          tableId: dto.tableId || null,
          orderNumber,
          type: dto.type,
          status: 'PENDING', // Awaiting payment verification
          subtotal: orderSubtotal,
          taxAmount,
          total: orderTotal,
          specialNotes: dto.specialNotes || null,
          orderItems: {
            create: orderItemsToCreate
          }
        },
        include: {
          orderItems: {
            include: {
              orderItemAddons: true
            }
          }
        }
      });

      // 7. Dispatch ticket to real-time Kitchen Displays
      if (dto.paymentMethod === 'CASH') {
        // Cash transactions go directly to KDS queue
        await tx.order.update({
          where: { id: order.id },
          data: { status: 'ACCEPTED' }
        });
        
        // Push notifications payload to branch screens
        this.kdsGateway.broadcastToBranch(dto.branchId, 'ticket.created', {
          ticketId: order.id,
          ticketNumber: orderNumber.substr(-3),
          priority: 'NORMAL',
          items: order.orderItems
        });
      }

      return {
        orderId: order.id,
        orderNumber,
        financials: {
          subtotal: orderSubtotal,
          taxAmount,
          total: orderTotal
        },
        status: order.status
      };
    });
  }
}
```
